Agent Registry — Completion Upgrade

Items 1–5 implemented in telemetry-included/awcp-mcp-temp-main/src/awcp/radar · documents what existed before and what was added · aligned to the AWCP magazine

Purpose. The agent registry (awcp.radar) already covered the magazine's core loop — register → quarantine → gate → degrade → observe. This upgrade closes the gaps that the registry itself owns, deferring the items that belong to other subsystems (approval tokens, the durable evidence ledger, replay/recovery, runtime adapters, Grafana dashboards). After this change the registry is "complete" as a standalone registry.

1 · Complete the quarantine rule

The magazine's Onboarding Quarantine says an agent leaves quarantine only after "telemetry, flag wiring, and policy callbacks … observed in execution." The registry previously checked only two of those three.

BEFORE   radar/onboarding.py · decide_status()

def decide_status(entry):
    missing = []
    if not entry.telemetry_enabled:
        missing.append("telemetry/observability")
    if not entry.policy_callbacks:        # only 2 of the magazine's 3 hooks
        missing.append("policy callbacks")
    if missing:
        return "quarantined", "missing control hooks: " + ", ".join(missing)
    return "active", None

AFTER

def decide_status(entry):
    missing = []
    if not entry.telemetry_enabled:
        missing.append("telemetry/observability")
    if not entry.feature_flags:           # ADDED — hook 2: "flag wiring" (magazine)
        missing.append("feature flags")
    if not entry.policy_callbacks:
        missing.append("policy callbacks")
    if missing:
        return "quarantined", "missing control hooks: " + ", ".join(missing)
    return "active", None
Knock-on change. Because the rule is now stricter, the four runtime agents had to declare a feature flag at registration or they would have fallen into quarantine. Their shared awcp_kit.register() now sends all three hooks (see the snippet under item 2).

2 · Per-agent risk tier → degradation budget

The magazine says "each workflow can override thresholds and ladders by risk." Previously the failure budget was a single global constant. Now every agent carries a risk tier that selects how many failures it tolerates before autonomy steps down — and an agent may still pin an explicit budget that wins over its tier.

BEFORE   radar/policy.py

# one global budget for everyone — could not vary "by risk"
FAILURE_BUDGET = int(os.getenv("AGENT_RADAR_FAILURE_BUDGET", "3"))

def budget_for(entry):
    return entry.failure_budget if entry.failure_budget else FAILURE_BUDGET

AFTER

DEFAULT_FAILURE_BUDGET = int(os.getenv("AGENT_RADAR_FAILURE_BUDGET", "3"))

# NEW — risk tier maps to a budget; env-tunable so NOTHING is hardcoded.
# AGENT_RADAR_RISK_BUDGET="low:5,medium:3,high:1" (tier names + values free)
RISK_BUDGET = _parse_risk_budget()      # default seed used only if env unset

def budget_for(entry):
    # precedence: explicit override → risk tier → system default
    if entry.failure_budget:
        return entry.failure_budget
    return RISK_BUDGET.get(entry.risk, DEFAULT_FAILURE_BUDGET)   # ADDED

The model gained one field (defaulting to "medium", so existing data is unaffected):

# radar/models.py — AgentEntry (and RegisterRequest)
risk: str = "medium"            # low | medium | high  (ADDED)
autonomy_ladder: list[str] = Field(default_factory=list)   # (already per-agent)
failure_budget: int | None = None

And the agents now declare the three hooks + their risk:

# agents/*/awcp_kit.py — register() payload
payload = {
    "telemetry_enabled": True,                          # hook 1
    "feature_flags": feature_flags or {"governed_writes": True},  # hook 2 (ADDED)
    "policy_callbacks": ["pre_write_gate", "on_signal"],     # hook 3
    "write_scopes": write_scopes,
    "risk": risk,                                       # ADDED (default "medium")
}

3 · Operator remove + CLI exclude

A real registry lets an operator forget an entry (hygiene), and should not list developer CLIs that merely embed an LLM SDK.

ADDED   radar/store.py — a removal method

def remove(self, agent_id):
    # operator action — drop an entry. A still-running scanned process
    # will simply be re-detected on the next scan; a stale entry stays gone.
    with self._lock:
        existed = self._entries.pop(agent_id, None) is not None
        if existed: self._persist()
        return existed

ADDED   radar/api.py — the DELETE endpoint

@app.delete("/agents/{agent_id}")
def deregister(agent_id):
    if not REGISTRY.remove(agent_id):
        raise HTTPException(404, "agent not found")
    _record_event("removed", agent_id, "operator removed entry")   # audit (item 5)
    return {"ok": True, "removed": agent_id}

ADDED   radar/detectors/base.py — exclude developer CLIs

EXCLUDE_HINTS = (
    "claude.app", "cursor.app", "vscode", "slack", "spotify", ...,
    # developer CLIs that embed an LLM SDK but are not governable agents
    "cloudcode_cli", "claude-code", "claude code",        # ADDED
)   # still extensible at runtime via the AGENT_RADAR_EXCLUDE env var

5 · Recent-decisions (audit) log

A registry-local, in-memory ring buffer captures the last N governance events so operators can see what the registry just did. This is explicitly not the durable Evidence Ledger (a separate subsystem) — it is the runnable stepping-stone toward it.

ADDED   radar/api.py — the buffer, recorder, and endpoint

from collections import deque

# in-memory ring buffer — bounded, newest-first, registry-local audit view
_EVENTS = deque(maxlen=int(os.getenv("AGENT_RADAR_EVENTS_MAX", "200")))

def _record_event(kind, agent_id="", detail="", **extra):
    _EVENTS.appendleft({"ts": time.time(), "kind": kind,
                        "agent_id": agent_id, "detail": detail, **extra})

@app.get("/events")
def events(limit: int = 50):
    return list(_EVENTS)[:limit]      # newest first

Recorders were threaded into the existing decision points (alongside the OTel counters that were already there):

# gate endpoint
_m_gate.add(1, {"decision": decision["decision"], "mode": decision["mode"]})  # existed
_record_event("gate", agent_id, f"{decision['decision']} ({decision['mode']})")  # ADDED

# signal endpoint — only when the ladder actually steps down
if result["degraded"]:
    _m_degrade.add(1, {"to": updated.autonomy_profile})                       # existed
    _record_event("degraded", agent_id, f"-> {updated.autonomy_profile}")   # ADDED

Scope note. Everything here lives inside the registry (awcp.radar) plus the agents' shared awcp_kit.py. The deferred items — expiring approval tokens, the durable evidence ledger, replay/recovery, runtime workflow adapters, and Grafana dashboards — are separate subsystems and were intentionally not built here. With items 1–5 the registry is complete as a standalone agent registry.
