01 — OPERATING MODEL

Operating Model for Governed Agent Ops

The control plane sits above existing runtimes. It registers agents, governs write actions, reduces autonomy safely when failures rise, and gives Agent Ops clear recovery paths.

Step 01
01

Register Agents & Attach Control Hooks

Register each agent with owner, runtime, declared write scopes, feature flags, telemetry, and policy callbacks. Under-instrumented agents stay visible, but they cannot execute governed write actions until onboarding is complete.

Agent Registry OTel Feature Flags
Step 02
02

Orchestrate Existing Workflows

Ingest active workflows from existing runtimes, normalize state, and preserve ownership and checkpoints across branches. The control plane does not replace the underlying execution stack.

Temporal Runtime Adapters Workflow DAGs
Step 03
03

Gate Write Actions

Evaluate every state-changing tool call against policy. High-risk actions pause until a narrow, expiring approval token is issued for the relevant workflow step and branch.

OPA Approval Tokens Tool Policies
Step 04
04

Degrade Autonomy Gracefully

At first failure signals, the control plane increases trace sampling, tightens retry and concurrency limits, shifts to safer profiles, and only then moves to recommendation-only before a hard stop. Each workflow can override thresholds and ladders by risk.

Trace Sampling Feature Flags Safe Profiles
Step 05
05

Replay & Recover

Use a replayable ledger with step history, context hashes, degradation state, and safe resume points so failed branches recover without restarting the full workflow.

Evidence Ledger Checkpoints Replay
Step 06
06

Generate Instrumentation Patches

For agents missing hooks, generate and attach a proposed patch or PR for telemetry, feature flags, and policy callbacks before they can leave quarantine.

Codex CLI Claude Code Patch Review
02 — AGENT OPS SCENARIOS

End-to-End Control Events

Each scenario follows one control event: trigger, control-plane response, safe outcome, and operator intervention. These are the core events for an Agent Ops team.

Scenario A Graceful Degradation
Trigger

Workflow Failure Budget Breach

Workflow refund-branch-882 records 3 failed write attempts, rising latency, and a stale handoff context between remediation steps.

Control Plane Response

Progressive Autonomy Reduction

Trace Sampler → increases capture depth
Policy Engine → tightens retry and concurrency limits
Safer Profile → pins a safer model and drops the branch to recommendation-only mode

Outcome

Service Preserved, Writes Narrowed

The workflow stays alive with no new writes. Agent Ops gets a replayable checkpoint, a narrowed action plan, and evidence for each autonomy reduction.

Operator Surface

Control Center + Incident Channel

Degradation state rises to the top of the home screen. Replay context and next-safe actions go directly to the operator and incident channel.

Scenario B Approval Tokens
Trigger

Cross-System Write Proposal

An agent proposes a bulk account update across CRM and billing APIs affecting 4,200 records and requiring a temporary privilege elevation.

Control Plane Response

Narrow Approval Request

Policy Engine → scopes the write set and risk tier
Approval Gate → bundles tool plan, context diff, rollback pointer, and expiry window
Operator → approves, denies, or escalates a branch-scoped token

Outcome

Single-Use Write Grant

The token authorizes only the approved action class for one branch and one expiry window. All other actions remain blocked, logged, and replayable.

Operator Surface

Approval Queue + Evidence Ledger

The operator reviews scope, checks prior failures and context hashes, then records the decision in the workflow ledger for replay.

Scenario C Onboarding & Quarantine
Trigger

Under-Instrumented Agent Discovered

New agent claims-triage-v1 appears through an existing runtime adapter with missing traces and no feature-flag callbacks.

Control Plane Response

Register, Quarantine, Patch

Registry → maps owner, runtime, and declared scope
Quarantine Policy → blocks governed write-capable work
Patch Generator → creates a Codex or Claude-assisted instrumentation patch for review

Outcome

Registered, Not Trusted Yet

The agent enters inventory right away, then leaves quarantine only after telemetry, flag wiring, and policy callbacks are merged and observed in execution.

Operator Surface

Agent Catalog + Patch Workflow

Agent Ops sees quarantine status in the workforce inventory while the owning team reviews the generated integration patch or PR.

04 — CONTEXT ENGINEERING

Multi-Turn Context Orchestration

Context engineering carries workflow state, policy outcomes, degradation state, and evidence across multi-agent execution. The control plane assembles only the minimum context needed to govern a branch, approve a write, or recover a failure.

INTAKE — INBOUND WORKFLOW SNAPSHOT

Workflow State Assembly

When an existing runtime emits or resumes a workflow, the Context Graph Manager builds governing context from runtime metadata, owner identity, recent handoffs, declared write scopes, active feature flags, and prior evidence entries. This becomes the initial control context for policy evaluation and operator visibility.

Context Graphs Agent Registry Runtime Adapters
GOVERNANCE — POLICY & APPROVALS

Policy-Gated Action Control

Temporal durable workflows govern the approval DAG. Each workflow step defines pre-conditions (risk signals, stale-context checks, failure budgets), execution policy (timeout, retry, idempotency keys, autonomy mode), and post-conditions (output validation, state assertions, ledger writes). High-risk or out-of-scope actions pause for a narrow approval token while the workflow waits durably without losing state.

Temporal Policy Engine Approval Gates
EXECUTION — SANDBOX & REMEDIATION

Sandboxed Actions and Patch Generation

When agents execute a tool call or generate missing control hooks, CodeAct provides the execution substrate. The agent returns Python or TypeScript artifacts in a sandboxed environment, including API responses, diff patches, trace captures, and patch proposals. The workflow stores these as compressed summaries and evidence entries.

CodeAct Modal Sandboxed Exec
RECOVERY — DEGRADATION & REPLAY

Graceful Degradation and Context Folding

As failures rise, the workflow gathers more context before reducing autonomy. Multi-signal triggers can increase trace sampling, tighten limits, switch to safer profiles, and move a branch to recommendation-only mode. A central degradation policy applies by default, while high-risk workflows can use local override ladders. Recursive Language Models (RLMs) compress large traces or documents, and the control plane writes summaries into a replayable evidence ledger with context hashes, degradation state, and resume pointers.

Degradation Policies RLMs Evidence Ledger Local LMs (scoring)

The recommended component architecture for governing an existing agent workforce in production, organized by functional layer and control boundary.

OPA + Flags + Local LMs Pentatonic SDK

09 — TECHNOLOGY STACK (RECOMMENDED)

Build & Deploy Stack

The technology stack is organized around three layers: control-plane primitives, partner platforms, and accelerators for governing existing agent runtimes.

Core Technologies
Temporal
Durable workflow execution engine for orchestration, safe resume, replay, retry policies, and long-running approval waits.
Context Graphs
Shared memory substrate for workflow state, context hashes, and recovery checkpoints across branches.
Agent Registry & Hooks
Versioned catalog of agent identity, owner, observability status, write scopes, feature flags, and policy callbacks. It is the admission point for governed workloads.
Policy Engine
Evaluates workflow steps and tool calls for approval tokens, scope checks, and autonomy reduction triggers. Central defaults can be overridden per workflow.
CodeAct
Agent-to-code execution framework that bridges reasoning and action for governed tool calls, replay analysis, and patch generation.
OpenTelemetry + Evidence Store
Trace capture, step outcomes, and replay artifacts for audit, quarantine checks, and recovery.
Platform Partners
Modal
Serverless compute for agent code execution, with isolated containers for controlled writes, replay tooling, and patch runs.
OpenFeature / Flipt
Feature-flag control plane for autonomy modes, workflow-path toggles, write-scope switches, and safe fallback policies.
Letta
Long-term memory management for agent and operator context. Useful for persistent workflow recall, ownership history, and retrieval-augmented recovery.
Anthropic / Claude
Primary reasoning provider for recovery planning, approval context generation, and high-fidelity patch proposals in governed workflows.
OpenAI / Codex
Code-centric reasoning provider for instrumentation patches, runtime adapters, and safe remediation tasks. A multi-provider strategy improves availability.
OSS Accelerators
Multica
Multi-agent conversation framework with structured communication protocols and shared context management during governed handoffs.
agent-session-bridge
Session continuity middleware for agent handoffs and runtime adapters. It ensures zero context loss during governed transitions.
graph-of-skills
Capability graph engine that models relationships between agents, scopes, and gaps in required control hooks.
Pentatonic ai-agent-sdk
Enterprise agent governance SDK with approval workflows, audit trails, degradation policies, and compliance hooks built in.
Codex CLI / Claude Code CLI
Patch-generation accelerators for under-instrumented agents. Produces proposed observability, feature-flag, and policy-hook integrations for owner review.
OpenClaw
LLM gateway with unified API, load balancing, failover, provider pinning, cost tracking, and caching across multiple providers.
