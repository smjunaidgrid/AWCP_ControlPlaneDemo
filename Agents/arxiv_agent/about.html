📚 arXiv Research Worker

An autonomous research worker that finds papers on arXiv and reports results — self-governed, and autonomously detected by the AWCP radar.

framework: LangGraph (ReAct) data: free arXiv API model: Ollama · llama3.1:8b port: 8103 folder: agents/arxiv_agent

What it is

A specialised autonomous worker for academic research. You give it a research goal; it queues the goal, then runs a multi-step LangGraph ReAct loop that searches the free arXiv API for relevant papers, optionally falls back to the web for general facts, synthesises an answer with citations, and routes its output through its own governed-write path (risk-tiered, with operator approval for high-risk external writes). Runs on a local Ollama model — no API keys.

Same self-contained worker engine as the other AWCP agents (queue, risk tiers, operator approval), and like them it is decoupled from AWCP (sends nothing; detected by scan) — what makes this one distinct is its arXiv tools and a research-oriented system prompt.

Architecture — how a goal flows

Features

  • Academic search — queries the free arXiv API for papers (title, authors, date, link, abstract) and fetches specific papers by id.
  • Task queue + autonomous worker — submit research goals; they're processed one at a time, multi-step.
  • Self-governed writes — risk-tiered; saving a bibliography or posting results pauses high-risk writes for operator approval (no control plane required).
  • Risk tiers + approval — save_artifact is a medium write; external_post is a high write that pauses for approval.
  • Deterministic finalize — the result is always persisted through the agent's own governed-write path.
  • Decoupled (autonomously detected) + task-console UI + free/local model.

How to run

  1. Model backend: 
    ollama serve
ollama pull llama3.1:8b      # first time only
  2. (Optional) control plane for live governance: 
    bash telemetry-included/awcp-mcp-temp-main/scripts/run_awcp.sh
  3. Start this worker (auto-creates its venv + installs arxiv on first run): 
    bash agents/arxiv_agent/run.sh
  4. Open → http://localhost:8103 and give it a research goal.

Or use the control panel: python3 agents/control_panel.py → http://localhost:8099.

Governance & its relationship to AWCP

This agent is decoupled from the AWCP control plane — it sends nothing to AWCP. Governance is split in two:

  • Agent-local (enforced here): risk tiers + an operator approval step that pauses high-risk external_post writes until approved on this agent's own POST /tasks/{id}/approve endpoint.
  • AWCP-side (observe-only): the radar detects this process autonomously by scanning it and lists it (as quarantined, since the agent declares no control hooks) — it can see the agent but does not gate, approve, or degrade it.
Note. arXiv calls hit the live, free arXiv API (rate-limited; be gentle). Tool-calling quality depends on the local model; the deterministic finalize guarantees the governed write regardless. Point ARXIV_MODEL at a stronger model for cleaner in-task tool use.
